# 04 · ID 编码规则速查

这张表里几乎所有"ID"都是把维度字段**拼接**出来的。下面把每个拼接型ID的规则整理成速查表，方便看到一个ID就能反推它代表什么。

> 共用维度缩写：
> **部位E**(1武器 2头饰 3衣服 4肩膀 5手套 6裤子 7鞋子 8副武器 9戒指 10耳环 11项链)、
> **职业G**(1战士 2弓 3法 4牧)、**转数H**(0~4)、**分支I/J**、
> **品质L**(1绿 2蓝 3普紫 4金 5橙 6远古 7太古)、**真实等级Q**、**转数等级R**、**物理魔法T**(1物2魔)。

---

## ★ 先理解：很多ID"既是关联字段、又是公式生成的"

这套表有个核心设计——**自洽编码（同一套ID规则跨表共用）**。像「使用职业限制」「初始基础属性库id」「垃圾库/高级库/远古库/太古库」「特殊属性库ID」这些字段，它们的值是指向别的表的**外键**，但**不用人去查、去填，而是用公式按规则拼出来**的。

**为什么能拼出来？** 因为目标表（equip_job_group / equip_proplib / equip_random_proplib / equip_special_drop）当初就是按**同一套编码规则**建ID的。equip 这边拿装备自己的维度（部位/职业/转数/品质…）一拼，拼出来的正好就是目标表里那条**早已存在**的ID。所以公式不是"凭空造ID"，而是"用规则反推出那个本就该有的外键值"。

- ✅ **好处**：策划不用每件装备都去翻关联表填ID，省事且不易填错。
- ⚠️ **风险（悬空外键）**：公式照样能拼出ID，但如果关联表里**没建**对应的那条（漏建了某个职业组/属性库），拼出来的ID就会"查无此项"，运行时出错。所以**改编码规则或新增维度时，关联表必须同步建好对应条目**。

> **对配置流程的意义**：正因为两侧同规则共生、equip 不读关联库，所以 **equip 可以先行**——它拼出的 ID 既是外键、又是查询键：拿去库里查，**命中即复用现有行、缺失即按同规则补建**（这就是 [10 文档](10_装备配置流程.md) 的"对账"步骤）。于是"复用现有库"和"新建库行"被统一成同一个动作的两个分支，不必纠结先后。

> 属于这种"公式生成的外键"的字段：使用职业限制(§3)、初始基础属性库id(§4)、垃圾库/高级库/远古库/太古库(§5)、特殊属性库ID(§7)。它们在 [05 文档](05_字段清单表.md) 里来源标"关联字段"；而把多个这种ID(加数量)打包成串的「初始随机属性库id」「装备技能随机库」标"关联组合"。它们的值都是公式拼的——和"是关联"不冲突，详见本节。

## 1. 种类编码（D列，装备ID基底）

```
种类 = "3" + 部位(2位) + 职业 + 转数 + 分支 + 品质 + 真实等级(3位)
```

| 段 | 来源 | 位数 |
|----|------|------|
| `3` | 固定前缀 | 1 |
| 部位 | E，补零2位 | 2 |
| 职业 | G | 1 |
| 转数 | H | 1 |
| 分支 | I | 1 |
| 品质 | L | 1 |
| 真实等级 | Q，补零3位 | 3 |

**例**：战士(1)·武器(01)·0转·分支1·绿(1)·1级 → `3` `01` `1` `0` `1` `1` `001` = **3011011001**

---

## 2. 装备ID（A列）

```
装备ID = 种类D （绝大多数行直接相等）
```
- **几乎所有行：装备ID = 种类(D/Remark1)。**
- **唯一例外**：当两件装备种类相同、会撞ID时，其中一行改用 `=种类+COUNTIF(...)` 错开+1。本表 206 行里仅 1 行这么做。

**例**：相邻两行种类都是 3011011001 → 一行=3011011001、另一行用 COUNTIF 错成 3011011002，避免撞号。

---

## 3. 使用职业限制ID（C列 → equip_job_group.GroupID）

```
职业限制 = INT(职业 + 转数 + "0" + 分支[1-2转])  [+10 若(转数<2 且 非武器)]  再接 3转分支
```

**例**：
- 武器·战士·0转·分支1 → `1`+`0`+`0`+`1`=1001，武器不+10，接J(1) → **10011**
- 头饰·战士·0转·分支1 → 1001，防具+10=1011，接1 → **10111**

**「防具+10」是什么意思**：+10 加在ID的**十位**上，把那一位（武器/防具区分位）从 `0`(武器) 变成 `1`(防具)。

| 位 | 千位 | 百位 | 十位 | 个位 |
|----|-----|-----|-----|-----|
| 含义 | 职业 | 转数 | **武器0/防具1** | 分支 |
| 武器 | 1 | 0 | 0 | 1 |
| 防具 | 1 | 0 | 1 | 1 |

目的是让防具指向**更宽松的防具职业组**——武器组能穿的职业少（只有用该武器的分支），防具组能穿的职业多。例：武器组 10011 含5个职业，防具组 10111 含8个职业。
**仅 0/1 转（转数<2）需要这个区分位**；2转及以上职业已细分，不再+10。

## 4. 初始基础属性库ID（X列 → equip_proplib.xlsx）

```
属性库ID = 物理魔法T + 部位(2位) + 转数 + 品质(2位) + 转数等级(3位)
```

**例**：物理(1)·武器(01)·0转·绿(01)·1级(001) → **101001001**

---

## 5. 初始随机属性库ID（Y列 → equip_random_proplib.xlsx）

格式：`库ID,条数;库ID,条数;…`（道具品质=2时为空）

```
= 垃圾库Z,(低级数量) ; 高级库AA,(高级数量) [; 远古库AB,1] [; 太古库AC,1]
```
- 垃圾库Z = `5 + 部位(2位) + "11"`（如 50111）
- 高级库AA = `5 + 部位(2位) + "15"`（如 50115）
- 远古库AB = `6 + 部位(2位) + "35"`（品质≥6）
- 太古库AC = `7 + 部位(2位) + "35"`（品质=7）

**各库抽几条（条数）** —— 垃圾库/高级库按品质查 Sheet4【6.稀有度/随机词条数量表】；远古/太古库固定每个 1 条：

| 品质 | 1绿 | 2蓝 | 3普紫 | 4金 | 5橙 | 6远古 | 7太古 |
|------|----|----|------|----|----|------|------|
| 垃圾库(低级) | 0 | 2 | 3 | 3 | 2 | 1 | 0 |
| 高级库 | 0 | 0 | 0 | 1 | 2 | 3 | 4 |
| 远古库 | — | — | — | — | — | 1 | 1 |
| 太古库 | — | — | — | — | — | — | 1 |
| **合计条数** | **0** | **2** | **3** | **4** | **4** | **5** | **5** |

## 6. 模型组ID（AF列，仅衣服E=3）

```
= "10" + 职业 + 转数段 + 分支     (转数0→H, 转数1→0, 转数>1→2)
```
**「转数段」是什么意思**：把转数归并成 3 档来控制模型复用——0转用 `0`、1转也用 `0`、2转及以上统一用 `2`。也就是说 0转和1转共用同一套模型，2转以后换一套。
其它部位的模型走 AG「挂件ID」（手填，可参照 Sheet4 BE~BL 清单）。

---

## 7. 特殊属性库ID（AH列 → equip_special_drop.xlsx）

仅当备注B 含"特效"字样才生成：
```
= "1" + 转数 + 部位(2位) + 件数修正(2位) + (职业&分支)(2位)
```
否则为空。

- **「仅特效才生成」**：公式先用 `SEARCH("特效", 备注)` 判断这件装备的备注里有没有"特效"两字，有才拼ID、没有就留空。
- **「件数修正」**：= IF(品质>4 且 部位=肩膀, 当前件数−3, 当前件数−1)。一般情况用"当前件数−1"，只有"高品质(>4)的肩膀"特殊用"−3"做偏移。

---

## 8. 装备类型（AO列，服务端）

```
= 部位E + 品质L + 真实等级Q + 当前件数V
```
**例**：武器·绿·1级·第1件 → **1111**

---

## 9. 图鉴套装ID（AR列）

仅当备注B 含"套装"字样才生成（用 `SEARCH("套装", 备注)` 判断备注里有没有"套装"两字，没有就留空）：
```
= 职业G + 转数H + 分支I + (当前件数V - 1)
```

## 10. 语言包Key（AI / AK列）

```
名称key = "EquipName_" + 装备ID
描述key = "EquipDes_"  + 装备ID
```
**例**：`EquipName_3011011002`、`EquipDes_3011011002` → 落到独立多语言文件 `language.xlsx`。

## 装备名称/描述的拼装规则（AJ / AL列）

不是纯ID，但同样是拼出来的：

```
装备名称 = 命名组词 + 部位词
         命名组词：按 命名组(由等级查得) × 转数 × 品质 在 Sheet4 词库取
                  （粗制旅人 / 曼荼罗 / 龙魂… 系列）
         部位词：按 部位 × 职业分支 在 Sheet4 GA~GJ 取（长剑/头盔/胸铠…）

装备描述 = 命名组词 + 描述前半句FS + 部位词 + 描述后半句FT
```

**例**：
- 名称：`粗制旅人` + `长剑` = **粗制旅人长剑**
- 描述：`最简单的` + `长剑` + `，在刚刚成为见习探渊者时，必备的装备。`

> 描述允许策划手写覆盖公式默认值。
