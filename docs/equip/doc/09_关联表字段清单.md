# 09 · 关联表字段清单

> equip 装备链各关联表的字段清单，格式与 [05](05_字段清单表.md) 一致：
> 序号 / 字段名称 / 字段类型 / 数据类型 / 字段名 / 来源 / 备注 / 关联表 / 关联字段。
>
> 字段类型：A 双端导出 / N 不导出(草稿) / S 仅服务端 / C 仅客户端。
> 来源四分类：关联字段（单个外键）/ 关联组合（多个外键±数量）/ 手工配置 / 公式生成。
>
> 分两部分：**一级（equip 直接关联）** 表 1~6；**二级（关联表的关联表）+ 通用语言表** 表 7~11。

**目录**

| #   | 表                          | 层级                               |
| --- | -------------------------- | -------------------------------- |
| 1   | equip_job_group            | 一级                               |
| 2   | equip_proplib              | 一级                               |

## 1. equip_job_group（职业组）

**作用**：定义"职业组"，用于转职后还能穿转职前职业的装备。**被 equip 的「使用职业限制(UseJobRequest)」引用**（equip.UseJobRequest = 本表 GroupID）。主表 Sheet1，44 行。

| 序号 | 字段名称 | 字段类型 | 数据类型 | 字段名 | 来源 | 备注 | 关联表 | 关联字段 |
|----|----------|------|--------|--------|------|------|--------|---------|
| 1 | 组ID | A | int | GroupID | 公式生成 | 职业组ID（本表主键，被 equip.使用职业限制引用）。由 基础职业(BaseJob)+转数(Remark2)+特殊定义(Remark3)+1-2转分支(Remark4)+3转分支(Remark5) 拼成 | | |
| 2 | 职业ID | A | string | JobID | 关联组合 | 该组包含的职业ID列表，多个用逗号隔开（如 1,1011001,1012001…）。每个ID是 soul 表的职业ID；本列由公式 TEXTJOIN 自动汇总符合条件的职业 | soul | SoulID |
| 3 | 职业名称 | C | int | ShowJob | 公式生成 | 装备tips下方客户端显示的"XXX职业"，取自本行「自身职业(Remark6)」 | | |

| 5 | 基础职业 | A | int | BaseJob | 手工配置 | 1战士 2弓箭手 3魔法师 4牧师 | | |
| 6 | 转数 | N | string | Remark2 | 手工配置 | 该组对应的转数 | | |
| 7 | 特殊定义 | N | string | Remark3 | 手工配置 | 组的特殊定义标记 | | |
| 8 | 1-2转分支职业 | N | string | Remark4 | 手工配置 | 1-2转分支编号 | | |
| 9 | 3转分支职业 | N | string | Remark5 | 手工配置 | 3转分支编号 | | |
| 10 | 自身职业 | N | string | Remark6 | 关联字段 | 该组的基准职业ID（值即 soul 的职业ID）；JobID/ShowJob 据它生成 | soul | SoulID |

## 12. item（道具表）

**作用**：全游戏道具总表。**装备和道具是配套关系——每配一件 equip 装备，就要在 item 表里配上对应的道具**（装备在背包/交易/掉落时，本质是以"道具"形态存在）。主表"道具"页 2434 行，另按职业拆了多个装备页签（见末尾"页签说明"）。

**与 equip 的对应**：装备道具的「道具ID(ItemId)」**就是 equip 的「装备ID(EquipID)」**（编码规则相同：3+部位+基础职业+转数+分支+品质+序号），名称语言包也共用同一个 Key。

### ★ 绑定 vs 非绑（核心差异）

一件装备在 item 表里要配**两条道具**——非绑版 + 绑定版，靠下面两个字段互相指向：

| | 非绑（可交易） | 绑定（绑角色，不可交易） |
|---|---|---|
| 道具ID(ItemId) | = 装备ID，如 `3011011001` | = "8" + 装备ID，如 `83011011001` |
| 绑定物品ID(BindItemId) | 填**绑定版**ID（83011011001） | 留空 |
| 未绑定物品ID(UnBindItemId) | 留空 | 填回**非绑版**ID（3011011001） |
| 所在页签 | 「战士装备」「弓箭手装备」… | 「战士装备-绑定装备」「…-绑定道具」 |

- 两条道具**绝大多数字段相同**（名称/品质/图标/出售返还…），只差 ID 前缀 和 BindItemId/UnBindItemId 这对指向。
- 玩家**绑定来源**（活动、签到等）发**绑定版**；**交易/掉落/商店**给**非绑版**。游戏靠 L/M 两字段在两者间转换（如非绑装备穿上后转成绑定版）。
- 配表时：在非绑页签配一条、在对应绑定页签配一条，ID 差个"8"前缀，L/M 对填即可。
