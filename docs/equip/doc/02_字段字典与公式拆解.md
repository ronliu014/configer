# 02 · 字段字典与公式拆解

逐列说明主数据页（"战士"页）的 55 列。每列给出：**列号 / 中文名 / 英文key / 类型 / 导出标记 / 单元格批注原文 / 公式翻译**。

公式统一以第 5 行为模板（实际每行把 `5` 换成本行行号）。`Sheet4!` 区块含义见 [03 文档](03_Sheet4参照数据字典.md)。

> 图例：🟢=策划手填　🔵=公式自动算　📤=会导出(A/S/C)　🗒️=草稿列(N,不导出)

## A · 装备ID `EquipID` · string · 📤A · 🔵
- **批注**：同道具ID
- **公式**：绝大多数行 `=D{行}`（直接等于 D 列「种类」）；**全表仅 R5 一行**例外，用 `=D5+COUNTIF($D$5:D5,D5)`。
- **翻译**：装备ID **基本就是「种类(Remark1)」**。COUNTIF 不是普遍机制，只是撞ID时的补丁：当两件装备种类相同会撞号时，其中一行用 `种类+COUNTIF` 错开 +1。
- **示例**：R5 与 R6 种类同为 3011011001 → R5 用 COUNTIF 错成 ID=3011011002、R6 直接 ID=3011011001，避免两行撞同一个ID。（实测 206 数据行中 205 行=种类，仅 R5 这一处错位）

## B · 备注 `Remark` · string · 🗒️N · 🔵
- **公式**：`=K5&H5&"转"&R5&"级"&M5&F5`
- **翻译**：拼一句人话备注 = 分支职业名 + 转数 + "转" + 转数等级 + "级" + 品质名 + 部位名。
- **示例**：`战士0转1级绿武器`。
- **额外用途**：此列文本还被 AH/AR 列用 `SEARCH("特效"/"套装",B5)` 反向判断该装备是否含特效/套装。

## C · 使用职业限制 `UseJobRequest` · int · 📤A · 🔵
- **批注**：对应 `equip_job_group` 的 GroupID，代表哪些职业可以直接穿这件装备
- **公式**：`=IF(AND(H5<2,E5<>1),INT(G5&H5&"0"&I5)+10,INT(G5&H5&"0"&I5))&J5`
- **翻译**：拼职业限制ID = 基础职业G + 转数H + "0" + 分支I，再接 3转分支J。当「转数<2 且 部位≠武器」时额外 +10（区分防具/武器的职业限制）。
- **示例**：武器 R5→`10011`；头饰 R8→`10111`。

## D · 种类 `Remark1` · string · 🗒️N · 🔵 ★ID基底
- **公式**：`="3"&TEXT(E5,"00")&G5&H5&I5&L5&TEXT(Q5,"000")`
- **翻译**：装备种类编码（SKU）= 固定前缀"3" + 部位(2位) + 基础职业 + 转数 + 分支 + 品质 + 真实等级(3位)。是 A 列装备ID 的来源。
- **示例**：`3`+`01`+`1`+`0`+`1`+`1`+`001` = `3011011001`。

## E · 装备部位 `EquipType` · int · 📤A · 🟢手填
- **批注**：1武器 2头饰 3衣服 4肩膀 5手套 6裤子 7鞋子 8副武器 9戒指 10耳环 11项链
- 策划手填的核心维度之一。

## F · 装备部位(名) `Remark2` · string · 🗒️N · 🔵
- **公式**：`=XLOOKUP(E5,Sheet4!$S$12:$S$22,Sheet4!$T$12:$T$22)`
- **翻译**：把部位编号 E 翻成中文名（查 Sheet4 部位表 S/T 列）。例：1→武器、2→头饰。

## G · 基础职业 `Remark3` · string · 🗒️N · 🟢手填
- 1战士 2弓箭手 3魔法师 4牧师。虽 key 是 Remark，但实际是手填维度。

## H · 转数 `TransferNum` · int · 📤A · 🟢手填
- **批注**：代表这件装备是几转可穿戴。

## I · 1-2转分支职业 `Remark6` · string · 🗒️N · 🟢手填
## J · 3转分支职业 `Remark28` · string · 🗒️N · 🟢手填
- 转职分支编号，参与种类/职业限制ID 拼接。

## K · 分支职业名称 `Remark7` · string · 🗒️N · 🔵【数组公式】
- **公式**：`=XLOOKUP(G5&H5&I5, Sheet4!$C$11:$C$99 & Sheet4!$D$11:$D$99 & Sheet4!$E$11:$E$99, Sheet4!$F$11:$F$99)`
- **翻译**：用「基础职业+转数+分支」三键拼接去 Sheet4 职业表匹配，取出分支职业中文名。例：1+0+1→`战士`，1+2+1→`迅剑士`，1+2+2→`盾剑士`。

## L · 品质 `Remark8` · string · 🗒️N · 🟢手填
- 1绿 2蓝 3普通紫 4金 5橙 6远古 7太古（对应 Sheet4 品质表 N/O 列）。

## M · 品质名称 `Remark9` · string · 🗒️N · 🔵
- **公式**：`=XLOOKUP(L5,Sheet4!N:N,Sheet4!O:O)`
- **翻译**：品质编号→品质中文名。1→绿，2→蓝，3→普通紫，4→金，5→橙，6→远古，7→太古。

## N · 道具品质 `Remark10` · string · 🗒️N · 🔵【外部引用】
- **公式**：`=XLOOKUP(L5,[1]Sheet1!$A$5:$A$18,[1]Sheet1!$B$5:$B$18)`
- **翻译**：品质→道具品质，查**外部工作簿** `[1]Sheet1`（`[1]` = `equip_rare.xlsx`，见 01 文档第5节）。

## O · 强度子品质 `Remark11` · string · 🗒️N · 🔵【外部引用】
- **公式**：`=XLOOKUP(L5,[1]Sheet1!$A$5:$A$18,[1]Sheet1!$D$5:$D$18)`
- **翻译**：品质→强度子品质，同样查外部工作簿 `equip_rare.xlsx`。

## P · 几级的装备 `Remark10` · string · 🗒️N · 🟢手填
- 同转数下"第几套"装备的序号，参与真实等级查表。

## Q · 真实等级 `RealLevel` · int · 📤A · 🔵【数组公式】
- **批注**：实际穿戴等级
- **公式**：`=XLOOKUP(H5&P5, Sheet4!AU:AU & Sheet4!AV:AV, Sheet4!AW:AW)`
- **翻译**：用「转数+第几套」两键拼接，去 Sheet4 真实等级表（AU转数/AV第几套/AW真实等级）查出真实等级。这是 Sheet4 里一张很长的展开表（AU~AX，行号到 310）。

## R · 转数等级 `Remark16` · string · 🗒️N · 🔵【数组公式】
- **公式**：`=XLOOKUP(H5&Q5, Sheet4!AU:AU & Sheet4!AW:AW, Sheet4!AX:AX)`
- **翻译**：装备上**显示给玩家看的等级（显示等级）**。拿「转数+真实等级」去 Sheet4 等级表（AU转数 / AW真实等级 / AX显示等级）查出对应的 AX 显示等级。
- **关键**：当前配表里 **AX(显示等级) 与 AW(真实等级) 数值完全相同**，所以转数等级算出来就等于真实等级(Q)。两列分开存，是为了将来能让"显示等级"独立于"真实等级"调整。
- **用途**：参与「备注(Remark)」和「初始基础属性库id(PropLibID)」的拼接。
- **附**：各转数的真实等级区间——转数0:1~5、转数1:6~50、转数2:51~100、转数3:101~200、转数4:201~300。

## S · 穿戴等级 `Level` · int · 📤A · 🔵
- **公式**：`=XLOOKUP(TEXT(H5,"0"),Sheet4!AU:AU,Sheet4!AW:AW,,0,1)`
- **翻译**：按转数取该转第一档的真实等级作为"最低穿戴等级"（XLOOKUP 第6参数=1 表示从头匹配首个）。

## T · 物理OR魔法 `Remark12` · string · 🗒️N · 🔵
- **公式**：`=IF(U5="物理",1,2)`
- **翻译**：物理=1，魔法=2。结果用于属性库ID X 的首位。

## U · 物理OR魔法备注 `Remark13` · string · 🗒️N · 🔵
- **公式**：`=XLOOKUP(K5,Sheet4!$F$11:$F$40,Sheet4!$G$11:$G$40)`
- **翻译**：按分支职业名查 Sheet4 得"物理/魔法"标签（F职业名→G攻击类型）。战士/弓=物理，法师/牧师=魔法。

## V · 当前件数 `Remark19` · string · 🗒️N · 🔵
- **公式**：`=IF(V_prev<W_prev, V_prev+1, 1)`（引用上一行的 V、W）
- **翻译**：套装内"第几件"的自增计数器，到达最大件数 W 后归 1，进入下一套循环。

## W · 最大件数 `Remark20` · string · 🗒️N · 🔵【数组公式】
- **公式**：`=IF(L5<5,1,INDEX(Sheet4!$AE$12:$AO$16, MATCH(H5,Sheet4!$W$12:$W$16,0), MATCH(E5,Sheet4!$AE$10:$AO$10,0)))`
- **翻译**：品质<5（橙以下）固定为1件；品质≥5时按「转数×部位」去 Sheet4 套数矩阵（AE12:AO16，行=转数 W12:W16，列=部位 AE10:AO10）查该部位最多有几套/几件。

## X · 初始基础属性库id `PropLibID` · int64 · 📤A · 🔵 ★
- **批注**：属性库id，与 `equip_proplib.xlsx` 对应
- **公式**：`=T5&TEXT(E5,"00")&H5&TEXT(L5,"00")&TEXT(R5,"000")`
- **翻译**：拼接 = 物理魔法T + 部位(2位) + 转数 + 品质(2位) + 转数等级(3位)。
- **示例**：`1`+`01`+`0`+`01`+`001` = `101001001`。

## Y · 初始随机属性库id `FixedPropLibID` · string · 📤A · 🔵 ★
- **批注**：随机词条库ID,数量；多个库用";"隔开。例：`50111,2;50135,1` 代表从库50111随机2条、50135随机1条。库ID取自 `equip_random_proplib.xlsx`
- **公式**：`=IF(N5=2,"", Z5&","&XLOOKUP(L5,Sheet4!$AZ$11:$AZ$17,Sheet4!$BB$11:$BB$17) &";"& AA5&","&XLOOKUP(L5,Sheet4!$AZ$11:$AZ$17,Sheet4!$BC$11:$BC$17) & IF(AB5="","",";"&AB5&",1") & IF(AC5="","",";"&AC5&",1"))`
- **翻译**：按品质组装"随机词条库+条数"字符串：
  - 道具品质N=2 时为空；
  - 否则 = 垃圾库Z,(低级库数量) ; 高级库AA,(高级库数量) ; 远古库AB,1 ; 太古库AC,1。
  - 各"数量"按品质查 Sheet4 稀有度表（AZ稀有度→BB低级数量/BC高级数量）。

## Z · 垃圾库 `Remark14` · string · 🗒️N · 🔵
- **公式**：`=5&TEXT(E5,"00")&"1"&"1"` → 普通词条低级库ID = 5 + 部位(2位) + "11"。例 `50111`。

## AA · 高级库 `Remark15` · string · 🗒️N · 🔵
- **公式**：`=5&TEXT(E5,"00")&"1"&"5"` → 普通词条高级库ID。例 `50115`。

## AB · 远古库 `Remark17` · string · 🗒️N · 🔵
- **公式**：`=IF(L5<6,"",6&TEXT(E5,"00")&"3"&"5")` → 品质≥6(远古)才有；ID = 6+部位+"35"。

## AC · 太古库 `Remark18` · string · 🗒️N · 🔵
- **公式**：`=IF(L5<7,"",7&TEXT(E5,"00")&"3"&"5")` → 品质=7(太古)才有；ID = 7+部位+"35"。

## AD · 战力类型 `CPType` · int · 📤A · 🔵
- **批注**：1公式 2数值
- **公式**：`=1`（恒为1，用公式算战力）。

## AE · 战力计算 `CPValue` · int · 📤A · 🔵
- **公式**：`=6` —— 单元格就是常数 6，不是计算式。
- **翻译**：配合「战力类型(CPType)」用的参数。CPType=1(走公式) 时，这里的 6 表示"用第6号战力公式/对应参数"；**真正的战力计算公式不在本表，由程序按此编号实现**。本表恒为 6。

## AF · 模型组id `AvatarGroupId` · int · 📤A · 🔵【数组公式】
- **批注**：服装使用模型组ID，武器使用挂件ID
- **公式**：`=IF(E5=3, IFS(H5=0,10&G5&H5&I5, H5=1,10&G5&0&I5, H5>1,10&G5&2&I5), "")`
- **翻译**：仅当装备部位(E)=3(衣服)才生成模型组ID（其余部位的模型走 AG 挂件ID）。ID = "10" + 职业(G) + 转数段 + 分支(I)；**转数段按转数(H)定：0转/1转→0、2转以上→2**。

## AG · 挂件ID `AccessoriesID` · string · 📤A · 🟢手填
- **批注**：服装使用模型组ID，武器使用挂件ID
- 手填资源ID，如 `11000103`。Sheet4 BE~BL 区块按转数/分支列了各职业各部位的挂件ID清单可供取值。

## AH · 特殊属性库ID `DropID` · int · 📤A · 🔵
- **批注**：代表特殊属性，从 `equip_special_drop.xlsx` 取
- **公式**：`=IF(ISNUMBER(SEARCH("特效",B5)), 1&H5&TEXT(E5,"00")&TEXT(IF(AND(L5>4,E5=4),V5-3,V5-1),"00")&TEXT(G5&I5,"00"), "")`
- **翻译**：`ISNUMBER(SEARCH("特效",B5))` = 在备注(B列)里搜"特效"两字，含"特效"才生成、否则留空。ID 由转数/部位/件数/职业分支拼成。

## AI · 装备名称语言包 `Remark23` · string · 🗒️N · 🔵
- **公式**：`=IF(AJ5="","","EquipName_"&A5)`
- **翻译**：有名称内容时生成语言包key `EquipName_<装备ID>`，给多语言文件 `language.xlsx` 用。

## AJ · 装备名称内容 `Remark24` · string · 🗒️N · 🔵【数组公式·最复杂】
- **公式（结构）**：
  ```
  IF(品质≥4 且 部位=武器,
     用「职业+转数+分支」匹配 Sheet4 武器专用命名组(DE15:DT15) → 取该命名组对应名,
     用通用命名组(CO15:DD15) 取名)
  & 部位后缀(查 Sheet4 GA/GC..GJ 各职业各部位的部位词，如 长剑/头盔/胸铠)
  ```
- **翻译**：自动拼装装备名 = 命名组词（按转数/品质/职业分级，如"粗制旅人""曼荼罗""龙魂…"）+ 部位词（长剑/头盔/胸铠…）。
- **示例**：`粗制旅人长剑`、`粗制旅人头盔`。命名组词库见 Sheet4 的 CK~DT、文案区。

## AK · 装备描述语言包 `Remark25` · string · 🗒️N · 🔵
- **公式**：`=IF(AJ5="","","EquipDes_"&A5)` → 描述语言包key `EquipDes_<装备ID>`。

## AL · 装备描述内容 `Remark26` · string · 🗒️N · 🔵/🟢
- **公式（数组）**：用「转数+品质+命名组」查 Sheet4 文案库（FS=描述前半句、FT=后半句）+ 部位词拼成整句。
- **翻译**：自动生成一句装备描述。例："最简单的长剑，在刚刚成为见习探渊者时，必备的装备。"
- **注意**：部分行策划**手写文案覆盖了公式**（如 R6 的"一般金属打造的粗制剑盾…"）。可见这是"公式给默认、策划可改写"的模式。

## AM · 装备图标 `Remark27` · string · 🗒️N · 🟢手填
- 图标资源名，如 `equip_w001_001` 或 `item_30210110101`。Sheet4 DG~DQ 区块列了各职业武器的 icon 资源名可供取值。

## AN · 初始套装库 `SuitId` · int · 📤A · 🔵
- **批注**：取 `equip_suit` 的ID，代表装备套装
- 多数行为空；有套装的行才填，规则见 AR。

## AO · 装备类型 `JobDropType` · int · 📤S · 🔵
- **公式**：`=E5&L5&Q5&V5` = 部位+品质+真实等级+当前件数。服务端用于掉落分类。
- **示例**：`1111`、`1151`。

## AP · 投放展示页签 `ShowTabId` · int · 📤C · 🟢
- **批注**：填0或不填则不在装备总览展示；ID参数看 `equip_show_tab`。**斗罗移植，目前无用**。

## AQ · 掉落等级 `JobDropLimitLevel` · int · 📤S · 🟢
- **批注**：**斗罗移植，目前没用到**。有对应代码，影响掉落。

## AR · 图鉴套装ID `HankBookSuitId` · int · 📤A · 🔵
- **公式**：`=IF(ISNUMBER(SEARCH("套装",B5)), G5&H5&I5&V5-1, "")`
- **翻译**：`ISNUMBER(SEARCH("套装",B5))` = 在备注(B列)里搜"套装"两字，含"套装"才生成图鉴套装ID（= 职业+转数+分支+(件数-1)）、否则留空。

## AS · 散件点数 `EquipPoint` · int · 📤A · 🔵
- **公式**：`=VLOOKUP(L5,Sheet4!$S$29:$T$35,2,FALSE)`
- **翻译**：按品质查 Sheet4「点亮积分设定」表（S稀有度→T积分），得该品质散件的点数。

## AT · 装备标签类型 `EquipHankType` · int · 📤A · 🔵
- **公式**：`=IF(L5=1,"",IF(E5=1,"2","1"))`
- **翻译**：品质(L)=1(绿)→无标签(空)；否则看装备部位(E)：=1(武器)→2、其余部位(非武器)→1。

## AU · 装备技能随机库 `RandonSkillID` · string · 📤A · 🟢手填
- 装备技能随机库ID列表，如 `10001;10002`。
