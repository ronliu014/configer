# equip 装备配置表 — 分析文档

> 源文件：`D:\SE\table\default_ios\equip\equip - 副本.xlsx`
> 分析时间：2026-06-22

本目录是对 **装备主表 `equip.xlsx`** 的逆向拆解文档。重点是把表里大量的 **单元格批注（备注）** 和 **公式** 拆出来、翻译成人话，让产品/策划在不打开 Excel 的情况下也能理解这张表的设计逻辑。

---

## 一句话总结

这是一张 **"半自动生成"** 的装备配置表：策划只手填少数几个"维度"列（部位、职业、转数、品质、第几套等），其余 **30+ 列全部由公式联动 `Sheet4`（参照字典页）自动算出**——包括装备ID、属性库ID、穿戴等级、装备名称、描述文案等。

理解这张表的关键 = 理解 **「哪些列是手填的、哪些是算出来的」+「ID 的编码规则」+「Sheet4 字典」** 三件事。

## 表的整体认知图

```
                  ┌─────────────────────────────────────────┐
   策划手填的维度  │  E部位  G职业  H转数  I/J分支  L品质  P第几套 │
   (输入)         └───────────────────┬─────────────────────┘
                                       │  公式联动
                                       ▼
                  ┌─────────────────────────────────────────┐
   Sheet4 字典页  │  职业表 / 品质表 / 部位表 / 真实等级表 /     │
   (参照)         │  命名组 / 文案库 / 套装数量表 / 模型icon表  │
                  └───────────────────┬─────────────────────┘
                                       │  XLOOKUP / IFS / INDEX-MATCH / 文本拼接
                                       ▼
                  ┌─────────────────────────────────────────┐
   自动算出       │  装备ID  种类  职业限制  穿戴等级  真实等级  │
   (输出)         │  属性库ID  随机词条库  装备名/描述/图标 ...  │
                  └─────────────────────────────────────────┘
                                       │  导出时只取 A/S/C 标记的列
                                       ▼
                              最终游戏可读配置
```

> ⚠️ 注意：表里大量 `Remark1`~`Remark28` 列是 **策划在 Excel 内部用来推导的"草稿列"**，导出标记为 `N`（不导出）。真正进游戏的字段只有约 20 列（标记为 `A/S/C`）。详见 01 文档。
