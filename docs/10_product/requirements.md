# 产品需求清单

状态：Active

本文记录 `configer` v1.0 及后续版本的产品需求。优先级含义：`Must` 为 v1.0 必须交付，`Should` 为 v1.1/v1.2 推荐交付，`Later` 为 v2.0 或更晚。

## 加载与会话

| ID | 优先级 | 需求 | 验收 |
|---|---|---|---|
| REQ-LOAD-001 | Must | 用户可以选择本地配表根目录。 | 未选择目录时不进入业务列表；选择后显示加载状态。 |
| REQ-LOAD-002 | Must | 加载 `equip`、`item`、`language` 和装备关联表。 | 加载结果显示成功表、缺失表和异常表。 |
| REQ-LOAD-003 | Must | 加载后建立本次会话 baseline。 | 变更预览能按 baseline 展示差异。 |
| REQ-LOAD-004 | Should | 记住最近一次目录句柄。 | 刷新后可提示恢复上次目录。 |

## 导航与信息架构

| ID | 优先级 | 需求 | 验收 |
|---|---|---|---|
| REQ-NAV-001 | Must | 左侧导航分为业务配置和公共配置。 | 装备位于业务配置，item 和 language 位于公共配置。 |
| REQ-NAV-002 | Must | 未加载时禁用业务入口。 | 用户不会进入空数据列表。 |
| REQ-NAV-003 | Later | 支持新增业务配置线。 | 新模块能按目录和导航规则接入。 |

## 装备列表

| ID | 优先级 | 需求 | 验收 |
|---|---|---|---|
| REQ-EQUIP-001 | Must | 显示装备列表和分页。 | 可查看装备 ID、备注、部位、职业、转数、品质、等级和状态。 |
| REQ-EQUIP-002 | Must | 支持搜索装备 ID 和备注。 | 输入关键字后列表只显示匹配项。 |
| REQ-EQUIP-003 | Must | 支持按职业、品质、转数、部位筛选。 | 筛选条件可组合并可重置。 |
| REQ-EQUIP-004 | Must | 支持进入新增、编辑、详情和删除确认。 | 每个操作都有明确入口和取消方式。 |

## 装备增删改

| ID | 优先级 | 需求 | 验收 |
|---|---|---|---|
| REQ-EQUIP-101 | Must | 新增装备时只允许编辑手填维度。 | 自动字段只读展示，不能手工修改。 |
| REQ-EQUIP-102 | Must | 系统根据维度展示生成预览。 | 装备 ID、关联 ID、语言 Key 随输入更新。 |
| REQ-EQUIP-103 | Must | 新增时检查装备 ID 冲突。 | 冲突时阻止保存并指向已有装备。 |
| REQ-EQUIP-104 | Must | 编辑时更新原行。 | 未修改 ID 时不会产生重复行。 |
| REQ-EQUIP-105 | Must | 删除装备需要二次确认。 | 确认前不改变内存数据。 |

## item 与 language

| ID | 优先级 | 需求 | 验收 |
|---|---|---|---|
| REQ-ITEM-001 | Must | 支持维护装备配套非绑和绑定两条 item。 | 详情中可查看并编辑两条道具。 |
| REQ-ITEM-002 | Must | 显示 item 互指关系。 | `BindItemId` 和 `UnBindItemId` 异常时给出提示。 |
| REQ-LANG-001 | Must | 支持维护装备和道具相关中文文案。 | 通过 Key 可新增或修改中文值。 |
| REQ-LANG-002 | Must | 引用 language 的字段优先显示中文。 | 未配置时显示未配状态和 Key。 |

## 关联查看与校验

| ID | 优先级 | 需求 | 验收 |
|---|---|---|---|
| REQ-REF-001 | Must | 关联字段可以打开只读抽屉。 | 抽屉显示目标表、目标主键和字段内容。 |
| REQ-REF-002 | Must | 校验关联主键是否存在。 | 存在显示正常，缺失显示悬空异常。 |
| REQ-REF-003 | Must | v1.0 不允许编辑关联表。 | 抽屉内没有保存、删除或补建操作。 |
| REQ-REF-004 | Should | 支持完整命中、缺失、异常三态对账。 | 列表可按配置状态筛选。 |
| REQ-REF-005 | Should | 支持关联表缺失补建。 | 缺失项可进入补建流程并写回关联表。 |

## 写回与记录

| ID | 优先级 | 需求 | 验收 |
|---|---|---|---|
| REQ-WRITE-001 | Must | 写回前展示变更预览。 | 用户能看到表、行、字段、旧值和新值。 |
| REQ-WRITE-002 | Must | 写回前生成备份。 | 备份失败时阻止写回。 |
| REQ-WRITE-003 | Must | v1.0 只写回 `equip`、`item`、`language`。 | 关联表不会被修改。 |
| REQ-WRITE-004 | Must | 只写允许修改的手填字段。 | 公式、格式、批注和非目标 sheet 保留。 |
| REQ-WRITE-005 | Must | 写回后生成 changelog。 | changelog 内容与变更预览一致。 |

## 非目标

| ID | 优先级 | 非目标 |
|---|---|---|
| NOGOAL-001 | Must | v1.0 不做自动 SVN 提交。 |
| NOGOAL-002 | Must | v1.0 不做 AI 辅助规则编写。 |
| NOGOAL-003 | Must | v1.0 不做批量导入。 |
| NOGOAL-004 | Must | v1.0 不做关联表编辑或自动补建。 |
| NOGOAL-005 | Must | v1.0 不把 demo 的 mock 逻辑作为业务规则。 |
