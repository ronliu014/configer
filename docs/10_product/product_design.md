# 产品设计

状态：Active

本文定义 `configer` v1.0 的产品形态。v1.0 以装备 `equip` 为首个业务配置线，目标是让策划在本地工具中完成装备、配套物品和文案的维护，并在写回 Excel 前看到明确的关联检查与变更预览。

## 产品定位

`configer` 的产品形态是“配置中心”：一个本地运行的 Web 工具，用统一入口管理不同配置任务。工具不要求用户固定使用仓库内 `source/` 目录，真实使用时由用户选择配表根目录。

v1.0 先完成装备配置闭环，后续再扩展到 `role`、掉落、技能、活动等业务配置线。

## UI 参考来源

[装备配置工具.html](../40_game_config/equip/demo/装备配置工具.html) 是产品同学设计的装备配置参考样例页面。后续实现应参考它的配置中心侧栏、装备列表、两阶段新增、详情双 Tab、关联抽屉、加载配表、写回预览和 changelog 展示。

该页面中的静态 mock 数据和简化计算逻辑不作为业务规则。字段、编码、关联、校验和回写规则以 `docs/40_game_config/equip/` 下的配置说明和工具开发实施指南为准。

## 用户角色

| 角色 | 关注点 | 主要操作 |
|---|---|---|
| 策划 / 配置维护者 | 快速增改配置、发现漏配和错配 | 选择配表目录，新增或编辑装备，维护 item 和 language，确认写回 |
| 开发支持者 / 工具维护者 | 规则准确、回写安全、模块可扩展 | 维护字段规则、页面能力、校验逻辑和回写约束 |

## 核心场景

- 新增一件装备，系统根据维度生成装备 ID 和关联字段预览。
- 编辑已有装备，检查配套 `item` 与 `language` 是否需要同步调整。
- 点击关联字段查看关联表条目；主键不存在时显示悬空异常。
- 写回前查看本次变更，确认备份和 changelog 后再落盘。

## 信息架构

```text
配置中心
├─ 业务配置
│  └─ 装备 equip
└─ 公共配置
   ├─ 物品 item
   └─ 文案 language
```

装备关联表不作为 v1.0 的独立导航入口。它们由装备详情、编辑页或关联抽屉进入，只读展示并参与存在性校验。

## 页面与入口

### 配表目录入口

首屏在未加载状态下提示用户选择配表根目录。加载成功后显示已加载表清单、加载时间和异常摘要。目录缺失或表缺失时，必须指出缺失路径和影响范围。

### 装备列表

装备列表是 v1.0 的主工作台。列表支持按职业 Tab、装备 ID / 备注搜索、品质 / 转数 / 部位筛选和分页浏览。列表操作包括新增、编辑、删除、查看详情。配置状态在 v1.0 只表达关联主键是否存在，不做完整链路三态对账。

### 装备新增 / 编辑

表单分为“手填维度”和“系统生成预览”。策划只编辑手填字段，系统生成字段只读展示。新增时如果装备 ID 冲突应阻止保存；编辑时允许保存当前装备自身 ID，并更新原行。

### 装备详情

详情页展示 `equip` 字段、配套 `item`、文案和关联字段。字段按手填、自动、关联三类呈现。关联字段点击后打开只读抽屉，支持查看目标条目和悬空异常。

### item 与 language 编辑

`item` 和 `language` 是装备闭环的公共配置入口。v1.0 允许从装备流程进入并编辑相关条目，也允许通过公共配置入口搜索和维护。装备引用处应优先显示中文文案，同时保留 Key 弱化展示。

### 写回预览

写回前展示变更预览，按表、行、字段列出新增、修改、删除。用户确认后工具先生成备份，再写回 `equip`、`item`、`language`，最后生成 changelog。

## 状态与提示

| 状态 | 展示要求 |
|---|---|
| 未加载 | 提示选择配表目录，不显示业务列表 |
| 已加载 | 显示表清单、加载结果和异常摘要 |
| 关联存在 | 关联字段可点击，抽屉显示目标条目 |
| 关联缺失 / 悬空 | 字段处标红，说明目标表和目标主键 |
| 可写字段 | 使用正常输入控件 |
| 只读字段 | 使用灰底或只读样式，不允许写回 |
| 待写回 | 顶部或底部显示未保存变更数量 |
| 已写回 | 显示备份位置和 changelog 入口 |

## v1.0 边界

- 只写回 `equip`、`item`、`language` 三类表。
- 装备关联表只读，不新增、不编辑、不补建。
- 校验只做引用主键存在性检查。
- 不做批量导入、图标资源库完整接入、AI 辅助规则编写和自动 SVN 提交。
- 产品样例页面只作为 UI 和交互参考，真实规则以 `docs/40_game_config/` 为准。
