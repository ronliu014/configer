# 策划操作流程

状态：Active

本文描述 `configer` v1.0 中策划完成装备配置的端到端流程。流程目标是让策划先完成装备、物品、文案三表维护，再通过只读关联校验确认是否存在悬空引用，最后安全写回 Excel。

## 主流程

1. 启动配置中心。
2. 在首屏选择配表根目录。
3. 工具加载 `equip`、`item`、`language` 和装备关联表。
4. 进入“业务配置 / 装备 equip”列表。
5. 搜索或筛选目标装备，或点击“新增装备”。
6. 在装备表单中填写手填维度，例如部位、职业、转数、品质、等级。
7. 系统展示装备 ID、关联 ID、语言 Key 等生成预览。
8. 保存装备，工具检查 ID 是否冲突。
9. 查看关联字段状态：主键存在则可下钻查看，主键缺失则显示悬空异常。
10. 编辑配套 `item`，维护非绑和绑定两条道具数据。
11. 编辑 `language` 文案，维护装备名称、描述和道具文案。
12. 打开变更预览，检查本次新增、修改、删除内容。
13. 确认写回，工具先生成备份。
14. 写回 `equip`、`item`、`language`。
15. 工具生成 changelog，用户据此走后续提交或人工检查流程。

## 新增装备流程

新增装备时，策划先填写装备维度。工具根据规则生成只读预览，包括装备 ID、物品 ID、语言 Key 和关联表引用 ID。保存时只校验装备 ID 冲突，不要求所有关联表都已补齐。

保存后，详情页应展示本装备的配套工作项：

- `equip` 主表行已创建。
- `item` 非绑与绑定道具是否存在。
- `language` 名称与描述 Key 是否存在。
- 关联表主键是否存在。

v1.0 对关联表只做“存在 / 不存在”提示，不提供补建入口。

## 编辑装备流程

编辑装备时，表单带入当前行数据。用户修改手填维度后，系统重新计算生成预览。保存时必须更新原装备行，不得误新增第二行。

如果修改导致装备 ID 变化，工具需要提示本次变更会影响 `item`、`language` 和关联引用。用户确认后再保存，并在变更预览中清楚列出受影响字段。

## item 与 language 流程

装备详情中展示配套 `item` 和 `language`。策划可以从装备上下文进入编辑，减少跨表查找成本。

- `item`：优先维护装备对应的非绑与绑定道具，并展示两者互指关系。
- `language`：输入中文文案；装备引用处显示中文，Key 作为辅助信息展示。

公共配置入口可用于搜索和维护已有 `item` 或 `language`，但 v1.0 不把它们扩展成完整独立平台。

## 关联查看流程

用户点击关联字段后，右侧打开关联抽屉。抽屉显示目标表、目标主键、存在性状态和只读字段列表。

如果目标条目存在，用户可以继续查看下一级关联。若目标条目不存在，抽屉显示“关联表无此条目”，并列出表名和主键。v1.0 不允许在抽屉内新增或编辑关联表数据。

## 写回流程

写回是用户主动操作，不能自动触发。工具必须在写回前展示变更预览，并说明将写回的表。

写回顺序：

1. 检查是否存在未确认的严重异常。
2. 生成原文件备份。
3. 逐单元格写回允许修改的字段。
4. 保留公式、格式、批注和非目标 sheet。
5. 生成 changelog。
6. 展示写回结果、备份位置和 changelog 入口。

## 失败处理

| 场景 | 处理方式 |
|---|---|
| 配表目录未选择 | 停留在未加载状态，提示选择目录 |
| 配表目录缺失必要表 | 显示缺失表名，禁用对应业务入口 |
| Excel 表头不符合协议 | 阻止加载该表，提示行号和预期协议 |
| 关联主键不存在 | 标记悬空异常，允许继续编辑三表 |
| 装备 ID 冲突 | 阻止新增，提示冲突装备 |
| 写回前备份失败 | 阻止写回，提示用户检查目录权限 |
| 写回失败 | 保留内存变更，提示失败表和字段 |
| 用户取消写回 | 不落盘，不生成 changelog |

## v1.0 只读边界

装备关联表在 v1.0 中只读。它们用于字段展示、引用选择和主键存在性校验；不支持新增、编辑、删除或自动补建。完整链路对账和关联表补建放入 v1.1。
