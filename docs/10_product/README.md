# 10_product 产品级文档

状态：Active

本目录保存产品设计，回答“策划如何使用工具完成工作”。产品文档应面向策划、产品、开发共同阅读，避免写成代码实现说明。

## 计划文档

- `product_design.md`：产品设计总稿，描述用户角色、核心场景、信息架构和主页面。
- `workflow.md`：策划操作流程，描述加载配表、编辑配置、检查链路、写回表格的闭环。
- `requirements.md`：需求清单，按版本和优先级记录功能点、验收口径和非目标。

## 编写原则

- 每个功能必须写清用户场景、入口、操作结果和失败提示。
- 页面设计要说明数据从哪里来、用户能改什么、哪些字段只读。
- MVP 文档必须优先服务 `equip`，但结构要能扩展到后续模块。
