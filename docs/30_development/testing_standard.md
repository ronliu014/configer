# 测试规范

状态：Active

本文定义 `configer` 的测试范围和验收重点。测试优先覆盖 Excel 安全、规则正确性和模块隔离，而不是只验证页面能渲染。

## 测试分层

| 层级 | 目标 | 示例 |
|---|---|---|
| 单元测试 | 验证纯函数和规则 | ID 编码、字段映射、diff 计算 |
| 集成测试 | 验证多个核心能力协作 | 加载 workbook → schema → baseline → diff |
| 回写测试 | 验证 Excel 文件安全 | 逐单元格写回、公式保留、备份生成 |
| UI 流程测试 | 验证关键策划路径 | 选择目录、装备列表、新增、变更预览 |
| 回归测试 | 防止历史问题复发 | `Remark` 撞名、整数和字符串匹配、非目标 sheet 丢失 |

## 命名规则

具体脚手架落地前，测试目录按目标代码镜像组织：

```text
tests/
  core/
  modules/
    equip/
```

JavaScript / TypeScript 使用 `*.test.ts` 或 `*.spec.ts`。Python 辅助脚本使用 `test_*.py`。

## 必测核心能力

### Excel 解析

- 能解析 4 行表头协议。
- 能保留源 sheet、源行号、源列坐标。
- 能识别手填列、公式列、关联字段和隐藏字段。
- 表头缺失或不匹配时返回结构化错误。

### schema 与字段 key

- 稳定逻辑 key 不依赖 `Remark` 名称。
- 同名源字段不会覆盖彼此。
- schema 与真实表头不匹配时禁止写回。

### ID 与关联校验

- 装备 ID 和关联 ID 生成结果与配置说明样例一致。
- 关联主键存在时返回目标条目。
- 关联主键缺失时返回表名、主键和触发字段。
- v1.0 不把缺失关联表自动补建为新行。

### diff 与 changelog

- 新增、修改、删除能被准确识别。
- 旧值和新值来自 baseline 与当前工作数据。
- changelog 与变更预览使用同一份 diff。

### 回写安全

回写测试必须覆盖：

- 只写 `manual` 字段。
- 公式列仍是公式。
- 单元格格式和批注保留。
- 非目标 sheet 保留。
- 写回前生成备份。
- 写回失败时保留内存变更并报告失败位置。

## UI 流程验收

v1.0 至少覆盖以下流程：

1. 未加载状态下业务入口不可用。
2. 选择配表目录后进入装备列表。
3. 搜索和筛选装备。
4. 新增装备并查看生成预览。
5. 编辑装备时更新原行。
6. 打开关联抽屉，查看存在和缺失两种状态。
7. 编辑配套 item 和 language。
8. 打开写回预览并取消写回。

## 测试数据

测试数据应使用最小化 fixture，不直接覆盖真实策划源表。需要真实 Excel 样例时，应复制到测试 fixture 或临时目录，并在测试结束后清理。

`source/` 是本地资源目录认知，不应作为自动化测试的唯一依赖。

## 完成标准

涉及 Excel 回写的变更，必须有回写测试。涉及编码或关联规则的变更，必须有至少一个历史样例或人工确认样例。涉及 UI 关键流程的变更，必须提供截图、录屏或自动化流程验证记录。
