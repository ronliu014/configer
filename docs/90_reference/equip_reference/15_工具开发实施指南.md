## 一、目标与范围

把"equip 装备配表"做成一个**本地配表编辑器**（"配置中心"的装备业务线）：

```
加载本地配表 → 在工具里增/改装备（维度输入 + 实时逻辑生成）→ 逐 FK 对账（命中/缺失/异常）
→ 补关联库 / item / 文案 → 写回本地配表（只改手填列、不碰公式）+ 生成变更记录
```

**本期范围**：equip 主表 + item + 7 张一级关联表（job_group / proplib / random_proplib / special_drop / suit / rare / skill_drop）+ 公共表（language / prop）。二级表（soul/prop/special_prop/skill_effect/skill）先做"只读查看下钻"。

**非目标（本期不做）**：技能子系统（buff/damage/param）、func/掉落/流程/天赋树等外围下游表、其它业务线（任务/活动配置）。

## 二、技术架构（推荐路线 · 已由 ItemEditor 验证）

同目录道具编辑器 `策划work\tools\ItemEditor` 已跑通同类架构，**直接沿用**（详见 [12 §13.7](12_装备配置工具设计规范.md)）：

| 层 | 选型 | 说明 |
|---|---|---|
| 形态 | 纯前端（单页 HTML，可后续模块化） | 无业务后端 |
| Excel 读写 | **SheetJS（xlsx.js）** | 浏览器端解析/生成 xlsx |
| 本地文件 | **File System Access API** | 直接读写本地 xlsx，原地回写；`file://` 受限，需静态 http 服务 |
| 句柄持久化 | **IndexedDB** | 记住"配表目录 / 图标目录"句柄，下次自动恢复 |
| 启动 | 一个静态 `http.server` + 可选图标服务 | 参考 ItemEditor 的 `start-all.bat` |
| 数据 | 全部内存（`allData` / `equipMap` / `langMap` / schema） | 不入后端、不生成中间 JSON |

> ⚠️ 本工具全是**本地文件操作、无外部 API**，不触发"外部 API 必须走飞书集成平台"的约束。
> **工具落地路径（已定）**：`D:\AI-PM-FLOW\策划work\tools\EquipEditor\`（与 ItemEditor 并列）。

**为什么 v1.0 没有"业务后端"（前端工程师即可）**：传统"读写 Excel"要后端，是因为浏览器以前碰不到本地文件、也解析不了 xlsx。本架构用 **File System Access API**（浏览器直接读写本地文件）+ **SheetJS**（浏览器内解析/生成 xlsx）+ **IndexedDB**（存句柄），把"加载 → 改 → 写回"整个数据闭环放进了浏览器，**不需要服务端逻辑、不需要数据库**。这里所谓"后端"只剩两样、且都不是业务后端：① 一个静态 `http.server`（一行命令，把 HTML 跑起来，因 `file://` 下 FSA 受限）；② 可选的图标图床（v1.0 用 FSA 直读图标目录即可，不需要）。

**何时才真需要本地后端**：① 跨浏览器——FSA 仅 Chromium 系（Chrome/Edge）支持，策划内部工具用 Chrome/Edge 即可，暂不涉及；② **v2.0 自动提交 SVN trunk / 对接 `excel-batch-commit`**——浏览器不能跑 `svn` 命令，那时需补一个轻量本地后端（Python）。**v1.0 写回到本地文件即止，提交仍走现有手动 / excel-batch-commit 流程，故纯前端足够。**

> 角色定位：v1.0 ≈ 前端 95% + 一个启动脚本（`.bat` / `http.server`）。用"前端工程师（能写少量本地启动脚本）"即可；到 v2.0 要做 SVN 自动提交时再引入后端/全栈。

## 四、功能模块与验收（M1~M10）

> 每个模块：**做什么** · 依据 · **验收点**。建议按 M1→M9 顺序，M10 后期。

### M1 · 配表加载
- FSA 选"配表根目录"→ SheetJS 读 equip + 关联表 + language/prop；按 **4 行表头协议**（Row1 中文名/Row2 A·N·S·C/Row3 类型/Row4 字段名/Row5+ 数据）解析，数据行记 `_rowIndex`。
- 建 `langMap={Id→Zhs}`（全表扫描）；建图标文件名索引（见 M5）；存目录句柄进 IndexedDB。
- 加载后建**基线快照**（深拷贝，回写 diff 用）+ 跑一次全量对账（M7）。
- 依据：12 §13.2 / §13.7。
- **验收**：选目录后列表出数据；语言字段显中文；刷新后目录句柄自动恢复；大表（proplib 2.3 万行）不卡死（懒加载/分页）。

### M2 · Schema 层（字段元数据驱动）
- 基础 schema **从 4 行表头自动生成**（`{key,srcCol,srcName,type,flag}`）；**叠加 13 的显式元数据**（control / source / ref / encode / enumSource / editable / format / status）。
- **字段 key 用稳定逻辑名，不用源表 Remark 名**（srcCol/srcName 仅作映射）——解决 Remark10 撞名（见 [14 §S3/Q2](14_工具化设计评审意见.md)、13 §二）。
- 依据：13 §二/§三、14 评审。
- **验收**：渲染/对账/回写全部按 `key` 索引，源表撞名/改名不影响工具。

### M3 · 装备列表页
- Tab（职业）+ 搜索（装备ID/备注）+ 筛选（品质/转数/部位/配置状态）+ 图标列 + **分页（每页 ~20，必须分页）** + 标记列（**只配置状态三态**：已配齐/待补 N/数据异常）。
- 依据：12 §5.1、§七。
- **验收**：万级数据流畅；标记列只显状态；筛选/搜索正确；循环内不用 `indexOf`（用 Map 索引）。

### M4 · 添加 / 编辑 / 批量导入
- 添加两阶段：填维度→实时逻辑生成（右栏）→保存落 equip 行→第2步配置补全清单（命中/缺失/异常 + 内联补建）。
- **编辑**：带入原维度，保存放行自身 ID、更新原行而非新增（demo 已示范，见 12 §5.2）。
- 批量导入：上传 Excel 多行→按编码批量生成→逐条进对账。
- 依据：12 §5.2、04 编码、13 encode。
- **验收**：新增撞 ID 拦截；编辑没改也能存；逻辑生成字段实时正确（与 04 一致）。

### M5 · 详情页 + 图标
- 整页 2 Tab（装备配置 equip / 道具配置 item）；字段**三态**（手填/自动/关联🔗）；**语言包字段显中文**（Key 弱化），点🔗下钻。
- 图标：FSA 选图标根目录→文件名（无扩展/小写）索引→`createObjectURL`；未命中回退"品质色底+部位首字"占位。资源路径参考 ItemEditor `D:\Art\Assets\UI\Texture\icon`。
- 依据：12 §5.3/§5.4、13 §3.5。
- **验收**：语言字段显中文不显 Key；图标命中显真图、未命中显占位。

### M6 · 关联抽屉（schema 驱动通用渲染）
- **一套通用渲染器**读 TableSchema 渲染任意表条目（替代 demo 的逐表 if-else）；`source=ref/combo` 字段挂🔗，按 `ref.kind`（single/group/combo/self）下钻；`group` 类（random_proplib/special_drop/skill_drop）渲染**整组带权重**；面包屑堆叠。
- 依据：13 §3.3、§四、§六；12 §六。
- **验收**：新增一张表只加 schema、不改渲染代码即可下钻；group 表显示整组。

### M7 · 对账引擎 + 配置状态
- 按 **encode（13 §3.4，以 04 为准）** 拼出每个 FK ID → 去库查：命中/缺失/异常（下游悬空 + 维度不一致）。
- 配置状态**写时算、读时取**（`cfgStatus/missingList/abnormalList/verifiedAt`），反向索引 `refIndex` 支持库变更增量重算；列表读字段不实时跑链路。
- **状态三态 + 优先级**：待补 > 数据异常 > 已配齐（见 [14 §S5](14_工具化设计评审意见.md)）。
- 依据：12 §十一、04、13 encode。
- **验收**：列表加载 O(1)/行；改库条目只重算受影响装备；三态与 missingList/abnormalList 一致。

### M8 · 回写配表（★ 红线最多）
- **逐单元格回写**：加载原 workbook，只 set 改动的 cell；**只写 `source=manual`**，公式列（formula/ref-gen/imported）一律不碰，保留公式/格式/批注。
- 新增装备=追加行只填维度、公式列向上复制；补库=对应 xlsx 追加行；改文案=写 language 的 Zhs。
- 写回前：**自动备份**（带时间戳）+ **ExcelMerge 比对确认**。多 sheet 必须**全部回写**。
- 依据：12 §13.3、§13.6、§13.7。
- **验收**：回写后用 Excel 打开，公式列仍是公式（不是静态值）；格式/批注保留；有备份；多 sheet 不丢。

### M9 · 变更记录（Changelog）
- 对 baseline 做 diff → 生成结构化记录（表→行ID→字段→旧值→新值，分新增/修改/删除）→ 落盘 changelog + 进"提交历史"。
- 作为提交说明对接 **`excel-batch-commit`** 提交 trunk。
- 依据：12 §13.4/§13.5；用户强调"写入即留痕"。
- **验收**：每次回写产出一份可读 changelog（见 12 §13.4 样例）；内容与实际改动一致。

### M10 · AI 辅助规则编写（可选 · 后期）
- 字段 AI 助手（自然语言↔encode/ref 双向）+ 用例验证器（填维度→拼 ID→对账）+ 一致性回写。
- 依据：13 §八。
- **验收**：能把自然语言规则译成可执行 schema 片段，并用样例对账验证。

## 五、关键约束与红线（必须遵守）

1. **回写只写手填列、公式列一律不碰**——这是工具能安全融入现有 Excel 配表流的前提（M8）。违反=毁掉配表可维护性。
2. **逐单元格回写，不用 `aoa_to_sheet` 整表重建**（ItemEditor 的硬伤，丢公式/格式）。
3. **字段 key 与源表名解耦**（稳定逻辑 key + srcCol/srcName），免疫 Remark 撞名/改名（14 §S3）。
4. **编码规则单一数据源**：生成与对账共用同一份 encode（13 §3.4，以 04 为准），杜绝两侧规则不一致→悬空外键。
5. **写回前必备份 + 必比对（ExcelMerge）+ 必出 changelog**。
6. **配置状态写时算、读时取**，列表不实时跑链路（性能，12 §十一）。
7. **多 sheet 全部回写**；**列表必分页**；循环内禁用 `indexOf`（用 Map 索引）。

## 六、避坑清单（ItemEditor 前车之鉴 · 详见 [12 §13.7](12_装备配置工具设计规范.md)）

- 整表重建写回 → 丢公式/格式/批注、多 sheet 丢数据 → **改逐 cell 回写**。

- 大表一次性渲染卡死 → **分页 + Map 预建索引**。
- Excel "整数 1 vs 字符串 '1'" 匹配偏差 → **统一类型**。
- 同名多文件改错版本 → 文件命名清晰、Grep 关键词定位。

## 七、版本拆分计划

> 按价值与依赖分版本交付。**v1.0 是最小可用闭环**：equip / item / language 三表增删改 + 关联表只读引用/校验 + 写回三表。关联表的编辑/补建、完整对账三态、AI 辅助等留后续版本。

### v1.0 · 三表增删改 + 关联只读引用（MVP · 当前优先）

**边界**：
- **可增删改**：equip、item、language 三张表。
- **关联表（job_group / proplib / random_proplib / special_drop / suit / rare / skill_drop / prop）= 只读**：显示关联数据、供引用选择；**主键不存在则报异常**（悬空外键）。本版**不编辑、不补建**关联表。
- **加载 / 回写**：equip + item + language（关联表只加载、不回写）。
- **对账简化**：只做"**引用主键存在性校验**"（存在→正常显示+下钻；不存在→标异常），**不做**完整命中/缺失/异常三态与配置状态性能架构（反向索引/写时算）。

**工作块**（括号 = 对应 §四模块的子集）：

| 块 | 内容 | 取自 |
|---|---|---|
| **V1-A 加载与基建** | 架构骨架（SheetJS+FSA+IndexedDB+静态服务）；加载 equip+item+language+关联表；4 行表头建 schema；基线快照；langMap | M1 + M2 |
| **V1-B equip 增删改** | 列表（筛选/搜索/分页）+ 添加（维度输入 + 逻辑生成显示 ID/FK）+ 编辑（带入维度、放行自身）+ 删除（二次确认）+ 详情（三态、关联🔗） | M3+M4+M5 |
| **V1-C item 增删改** | item 列表/详情；配套两条（非绑+绑定，BindItemId/UnBindItemId 互指）；增删改 | M3+M4+M5 |
| **V1-D language 增删改** | 文案表界面（Key + 中文 + 搜索）；增删改；equip/item 引用处**显中文** | M5（语言部分） |
| **V1-E 关联引用与校验 ★** | 关联字段：主键存在→显示数据 + 只读下钻、引用选择（ref-picker）；**主键不存在→标「⚠ 关联表无此条目」异常**；关联抽屉**只读**（schema 驱动通用渲染）；group 类（随机库/特效/技能）显整组 | M6（只读）+ M7（仅存在性校验） |
| **V1-F 回写 + 变更记录** | 回写 equip/item/language（逐 cell、只写手填列、不碰公式、写前备份）；diff baseline 出 changelog | M8 + M9 |

**v1.0 明确不做**：关联表编辑 / 缺失补建、完整对账三态 + 配置状态写时算/反向索引、图标资源库（占位即可）、批量导入、AI 辅助、自动提交。

**v1.0 验收**：加载真实配表 → 增/改/删 一件装备及其 item 两条道具、改其文案 → 引用的关联表主键存在则显数据、缺失则标异常 → 写回三表：Excel 打开公式列仍是公式、有备份、changelog 准确。

### v1.1 · 对账完善 + 关联表补建
- 完整对账三态（命中/缺失/异常，含下游悬空 + 维度不一致）+ 配置状态写时算 + 反向索引（M7 全量）。
- 关联表增删改 / **缺失补建**（对账的补建分支）→ 回写扩展到关联表。
- 配置状态三态进列表（待补 / 数据异常）。

### v1.2 · 体验与规模
- 图标资源库接入（FSA 图标索引，M5 完整）；批量导入（M4）；性能（分页/Map 索引/大表懒加载）；特效/套装/技能完整链编辑。

### v2.0 · AI 辅助 + 平台化
- AI 辅助规则编写（M10，13 §八）；衔接 `excel-batch-commit` 自动提交；多业务线接入（任务/活动配置）。

> **依赖关系**：V1-A 是一切基础；V1-E 的"存在性校验"是 v1.1 完整对账的前身；v1.0 回写只动三表，v1.1 才扩到关联表。

## 九、demo 的角色（务必理解）

demo 是**页面样式 / 交互蓝本**：列表、添加两阶段、详情双 Tab + 三态、关联抽屉、批量添加、加载/写回入口（含变更记录预览）——**都已用静态 mock 呈现"长什么样"**。开发时**照它的视觉与交互**，但**逻辑（编码/对账/加载/回写/schema）一律以文档为准**，不要从 demo 反推逻辑（demo 的编码函数是简化 mock，与 04 不完全一致）。详见 [12 §十二](12_装备配置工具设计规范.md)。
