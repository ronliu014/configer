# 05 · equip 装备表 · 字段清单

> 源文件：`equip.xlsx` · 主数据页（战士页）共 55 列
> 列结构：序号 / 字段名称 / 字段类型 / 数据类型 / 字段名 / **来源** / 备注 / 关联表 / 关联字段

**字段类型说明**

| 标记 | 含义 |
|------|------|
| A | 双端导出（客户端+服务端都用） |
| N | 不导出，只是策划在 Excel 里用来推算的"草稿列"（字段名多为 Remark） |
| S | 仅服务端用 |
| C | 仅客户端用 |

**「来源」列说明**（四选一）

| 值 | 含义 | 关联表/关联字段列 |
|----|------|-----------|
| 关联字段 | 该字段的值是指向另一张表/外部文件的**单个**ID（跨表外键），哪怕由公式拼出 | **填**（关联表写表名、关联字段写目标列） |
| 关联组合 | 一个字段里**组合了多个**关联ID（可再附数量/参数），如 `库ID,条数;库ID,条数`、`道具ID,数量`、`ID;ID`；值可由组成字段推导 | **填**（写内部嵌的关联表与字段） |
| 手工配置 | 策划手动录入的"输入维度" | 不填 |
| 公式生成 | 由其它列计算/查表自动算出（非关联、非组合） | 不填 |

> 规则：「关联字段」「关联组合」要填"关联表 + 关联字段"两列；手工配置 / 公式生成两列留空，说明写在备注里。

**「关联字段」vs「公式生成」怎么分**（最易混的一对）——核心不是"是不是公式"（两者都可能是公式算的），而是看**字段的值指不指向另一张独立配置表的主键**：

| | 关联字段 | 公式生成 |
|---|---------|---------|
| 值的本质 | 一个**指针**＝另一张独立表的主键ID | 一个**结果值**＝名字/等级/标志/内部码 |
| 游戏怎么用 | 拿它去那张表查数据 | 直接用，不再查别表 |
| 是公式拼的也算 | ✅ 仍是关联字段（"自洽编码"） | — |

判别三看：① **批注**写"对应/指向/取自 XX表" → 关联字段；② **值的形态**（库ID/组ID=指针 → 关联；中文名/等级 → 公式）；③ 查的是**表内 Sheet4 字典**(翻译)算公式生成，指向**独立 equip_xxx 表**才算关联字段。
> 例：使用职业限制（公式拼出 GroupID、指向 equip_job_group）= **关联字段**；分支职业名称（查 Sheet4 得"迅剑士"）= **公式生成**。

## 怎么读这张表（给第一次看的人）

按「来源」列分四类：

- **关联字段（13 项）**：值是指向别表/外部文件的**单个**ID，建飞书多维表格时这些就是要建"关联"的字段——
  使用职业限制→equip_job_group、初始基础属性库id→equip_proplib、垃圾库/高级库/远古库/太古库→equip_random_proplib、特殊属性库ID→equip_special_drop、初始套装库→equip_suit、投放展示页签→equip_show_tab、道具品质/强度子品质→equip_rare、装备名称语言包/装备描述语言包→language。
- **关联组合（2 项）**：① 初始随机属性库id——"库ID,条数;…"，由垃圾库/高级库/远古库/太古库这 4 个关联ID 加各自条数打包而来；② 装备技能随机库——"技能组ID;技能组ID"，多个关联ID组合，关联 equip_skill_drop（再经 EquipSkillEffectID 链到 equip_skill_effect）。第①项关联 equip_random_proplib。这类"多关联ID(可带数量)"格式在很多表里都有（如 道具ID,数量）。
- **手工配置（9 项）**：策划真正手动录入的输入维度——装备部位、基础职业、转数、1-2转分支职业、3转分支职业、品质、几级的装备、挂件ID、装备图标（外加遗留字段掉落等级）。
- **公式生成（其余）**：把上面的手填值算成中文名、等级、各种非关联ID等。

其它说明：

- **"关联字段"也可能是公式拼出来的，不矛盾**：使用职业限制、初始基础属性库id、垃圾库/高级库/远古库/太古库、特殊属性库ID 这些，既是指向别表的外键，值又由公式按规则拼成——因为关联表用了同一套编码规则，公式拼出的正好是对方已存在的ID（叫"自洽编码"）。原理、好处和"悬空外键"风险见 [04 文档开头](04_ID编码规则速查.md)。
- **"关联组合"是"关联字段"的复合版**：把多个关联ID连同数量/参数打包成一串（库ID,条数;…）。它内部的单个关联ID（如垃圾库/高级库）本身就是关联字段，整串则由公式推导，不用手填。
- **备注里说"去 Sheet4 查"的字段**（部位名、分支职业名、品质名、真实/穿戴等级、最大件数、装备名/描述、散件点数等），它们的查表逻辑——拿什么当钥匙、查到什么、例子——都汇总在 [03 文档开头的"速览表"](03_Sheet4参照数据字典.md)，一看就懂，不用啃坐标。
- **字段名是 Remark 的列基本都不导出**（字段类型=N），只是 Excel 内部的计算中转站；真正进游戏的是字段类型 A/S/C 的那 21 列。
- 整体跨表关系见 [08 ER关系图](08_ER关系图.md)。
- 第 48~55 列在主数据页是空列，没有字段，未列入。
