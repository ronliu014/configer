# 40_modules 模块级文档

状态：Active

本目录保存各业务模块的文档入口和摘要。模块文档用于连接项目级设计与具体领域资料，避免开发者直接在大量历史分析文档中迷路。

## 当前模块

- [equip](equip/README.md)：装备配置 MVP，当前首个业务线。

## 计划模块

- `item/`：物品与装备配套道具能力。
- `language/`：多语言文案表能力。
- `role/`：角色基础配置能力。
- `drop/`：掉落库与奖励配置能力。

## 模块文档职责

每个模块至少应有 `README.md`，说明模块定位、范围、依赖表、关键规则、开发入口和验收标准。模块详细规则可以链接到 `docs/equip/doc/` 或未来的领域资料目录，但模块入口必须给出阅读顺序和当前有效口径。
