# 术语表

状态：Active

本文统一 `configer` 项目中的核心术语。后续项目说明、产品设计、技术方案、模块文档应优先使用这里的名称。

## 项目与产品

| 术语 | 说明 |
|---|---|
| configer | DD 游戏项目的本地策划配置工具，用于自动化生成、检查和写回 Excel 配表。 |
| 配置中心 | configer 的产品形态名称，表示承载多条业务配置线的统一工具界面。 |
| 业务配置线 | 一个相对独立的配置业务域，例如 `equip`、`item`、`role`、`drop`。 |
| 公共配置层 | 被多条业务配置线共享的基础配置，例如 `item`、`language`、`prop`、`skill`。 |
| MVP | 最小可用版本。当前指装备配置 v1.0 闭环。 |

## Excel 与配表

| 术语 | 说明 |
|---|---|
| 配表 | 策划维护的 Excel 配置表，最终被游戏读取或导出为游戏配置。 |
| 配表根目录 | 用户在工具中选择的本地配置目录，通常包含 `equip/`、`item/`、`language/` 等子目录。 |
| 主表 | 一个业务配置线的核心表，例如装备业务线的 `equip.xlsx`。 |
| 关联表 | 被主表引用的配置表，例如装备属性库、职业组、随机词条库、套装库。 |
| 公共表 | 跨业务线共享的配置表，例如 `item`、`language`、`prop`。 |
| 4 行表头协议 | Excel 前 4 行分别表示中文名、导出标记、类型、字段名；第 5 行起为数据。 |
| A/N/S/C 标记 | 配表导出标记。具体含义以游戏配置说明中的表结构文档为准。 |

## 字段与规则

| 术语 | 说明 |
|---|---|
| 手填列 | 策划直接维护、工具允许写回的字段。schema 中通常标记为 `manual`。 |
| 公式列 | Excel 公式生成的字段，工具不得写成静态值。 |
| 关联字段 | 指向另一张表主键或条目的字段，通常支持下钻查看。 |
| 关联组合 | 多个字段共同组成一个关联含义或引用链路。 |
| schema | 工具使用的表/字段元数据，描述字段 key、来源、控件、引用和编码规则。 |
| 稳定逻辑 key | 工具内部使用的字段标识，不直接依赖 Excel 中的 `Remark` 名称或中文列名。 |
| srcCol / srcName | Excel 原始列坐标和源列名，用于回写定位和源表映射。 |
| ID 编码规则 | 由多个维度字段拼接或计算出业务 ID 的规则，例如装备 ID、属性库 ID。 |

## 校验与对账

| 术语 | 说明 |
|---|---|
| 对账 | 按规则生成或读取引用 ID，并检查目标表是否存在、是否一致、是否异常。 |
| 悬空引用 | 主表引用了不存在的目标条目。MVP 中通常表现为“关联表无此条目”。 |
| 缺失 | 按配置规则应该存在但当前没有的关联条目，后续版本可引导补建。 |
| 数据异常 | 目标条目存在，但内部字段、下游引用或维度与当前配置不一致。 |
| 已配齐 | 对账项全部命中且没有异常。 |
| 待补 | 存在缺失项，优先级高于数据异常。 |
| 配置状态 | 条目的整体校验状态，规划为 `已配齐`、`待补`、`数据异常`。 |

## 数据生命周期

| 术语 | 说明 |
|---|---|
| 加载 | 选择本地配表目录，读取 Excel workbook 并解析为工具内存数据。 |
| baseline | 加载后保存的原始快照，用于后续 diff 和 changelog。 |
| diff | 当前内存数据相对 baseline 的差异。 |
| 回写 | 将工具中的变更写回本地 Excel。必须逐单元格写入，只写手填字段。 |
| 备份 | 写回前复制原始文件，防止误写后无法恢复。 |
| changelog | 一次写回产生的变更记录，包含表、行、字段、旧值、新值。 |

## 技术与运行

| 术语 | 说明 |
|---|---|
| File System Access API | 浏览器访问本地文件和目录的能力，MVP 用于选择并写回配表。 |
| SheetJS | 浏览器端读取和写入 Excel 的库。 |
| IndexedDB | 浏览器本地存储，用于保存目录句柄和用户偏好。 |
| 静态服务 | 用于启动本地 Web 工具的 HTTP 服务，避免 `file://` 访问限制。 |
