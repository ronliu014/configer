# 阶段路线图

状态：Active

本文定义 `configer` 的阶段能力边界。路线图用于控制范围，避免在 MVP 阶段同时展开过多配置模块和平台能力。

## 总体原则

- 先做装备业务线闭环，再扩展其它业务线。
- 先保证 Excel 加载、校验、回写安全，再增强 UI 和自动化。
- 先做本地工具，不引入业务后端。
- 先沉淀可执行规则，再引入 AI 辅助规则编写。

## v1.0 装备 MVP

目标：完成装备配置的最小可用闭环。

范围：

- 加载本地配表目录中的 `equip`、`item`、`language` 和装备相关关联表。
- 支持 `equip` 装备表增删改。
- 支持 `item` 配套道具表增删改。
- 支持 `language` 文案表增删改。
- 支持装备关联表只读查看、引用选择和主键存在性校验。
- 支持详情页字段三态：手填、自动、关联。
- 支持写回 `equip`、`item`、`language`。
- 写回前生成备份，写回后生成 changelog。

明确不做：

- 不编辑或补建装备关联表。
- 不做完整链路对账三态和反向索引。
- 不做图标资源库完整接入。
- 不做批量导入。
- 不做 AI 辅助规则编写。
- 不做自动 SVN 提交。
- 不接入其它业务线。

验收标准：

- 能加载真实装备相关配表并进入装备列表。
- 能新增、修改、删除一件装备及其配套 item 和 language。
- 关联表主键存在时能展示和下钻，不存在时能标记异常。
- 写回后 Excel 公式、格式、批注、多 sheet 保留。
- changelog 能准确描述本次变更。

## v1.1 对账完善与关联补建

目标：把“只读存在性校验”升级为完整配置链路对账。

范围：

- 支持命中、缺失、数据异常三态。
- 支持 `cfgStatus`、`missingList`、`abnormalList`、`verifiedAt` 等状态字段。
- 支持写时算、读时取的配置状态策略。
- 支持反向索引，关联表变更时只重算受影响装备。
- 支持装备关联表增删改和缺失补建。
- 写回范围扩展到装备关联表。

明确不做：

- 不做多业务线平台化接入。
- 不做 AI 辅助规则编写。
- 不做 SVN 自动提交。

验收标准：

- 列表能按配置状态筛选。
- 修改关联表条目后，只刷新受影响装备状态。
- 缺失项能从待补清单进入补建流程。
- 数据异常能说明原因，例如下游悬空或维度不一致。

## v1.2 体验与规模优化

目标：提升策划使用效率和大表处理能力。

范围：

- 接入图标资源目录，命中显示真实图标，未命中使用占位。
- 支持批量导入装备维度数据。
- 优化大表加载、索引和分页。
- 完善特效、套装、装备技能等更深配置链路的编辑体验。
- 强化变更预览和提交历史查看。

明确不做：

- 不改变 v1.0/v1.1 的安全回写原则。
- 不把所有公共表做成完整独立平台。

验收标准：

- 大表列表和关联抽屉操作保持流畅。
- 批量导入能逐条展示成功、缺失和异常。
- 图标资源缺失不会阻塞配置流程。

## v2.0 AI 辅助与平台化

目标：从装备工具扩展为多业务线配置中心。

范围：

- 支持 AI 辅助规则编写，将自然语言规则转为 schema、encode、ref 配置草案。
- 支持规则用例验证，输入维度后展示生成 ID、关联引用和对账结果。
- 接入更多业务线，例如 `role`、`drop`、`skill`、活动配置。
- 评估轻量本地后端，用于 SVN 自动提交、ExcelMerge 调用和批处理流程。
- 对接现有提交流程，把 changelog 转为规范提交说明。

明确不做：

- 不允许 AI 直接覆盖已确认规则。
- 不允许没有人工确认的 AI 规则进入 Active 文档或生产 schema。
- 不破坏本地 Excel 配表工作流。

验收标准：

- 新业务线可通过配置说明和 schema 接入，不需要重写平台框架。
- AI 生成的规则必须能通过样例验证。
- 自动提交流程可追溯、可回退、可人工确认。

## 跨版本红线

- 公式列不得写成静态值。
- 不使用整表重建方式覆盖 workbook。
- 写回前必须备份。
- 写回后必须生成 changelog。
- 生成和对账必须共用同一套编码规则。
- 业务模块必须隔离，修改一个模块不得破坏其它模块。
- demo 只作为交互参考，不作为真实业务逻辑来源。
