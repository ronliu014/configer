# 00_project 项目级文档

状态：Active

本目录保存项目级决策，回答“为什么做、做到什么范围、按什么阶段推进”。这些文档不写页面细节和代码细节。

## 当前文档

- [项目说明书](project_overview.md)：描述背景、目标、用户、范围、非目标和成功标准。
- [术语表](glossary.md)：统一配置中心、配表、对账、手填列、公式列、关联库等名称。
- [阶段路线图](roadmap.md)：定义 v1.0、v1.1、v1.2、v2.0 的能力边界。

## 编写原则

- 项目级文档只写长期稳定的方向和边界。
- 版本计划必须说明“本期做什么”和“本期不做什么”。
- 任何影响多个模块的决策，应优先沉淀到本目录。
